Orientações sobre os Trabalhos de Programação

Orientações e gerenciamento dos trabalhos de programação.

Data
Aug 1, 2016
Autor
Ricardo C. Corrêa (corre.nosp@m.a@uf.nosp@m.rrj.b.nosp@m.r)

Estas são instruções gerais para trabalhos práticos de programação. Notação:

  1. [Y]: número do trabalho. Deve ser substituído por 1 para o trabalho 1, 2 para o trabalho 2, e assim por diante.
  2. [LABGADIR]: diretório para abrigar os arquivos dos trabalhos. Deve ser substituído pelo nome de escolha pessoal para abrigar os arquivos relativos aos trabalhos de Laboratório de Grafos e Algoritmos.
  3. [ARCH]: arquitetura da máquina utilizada. Deve ser substituído por 32 ou 64, dependendo da arquitetura utilizada.

Arquivos Fornecidos

Além da descrição disponível na aba Arquivos, os seguintes arquivos são fornecidos em trab[Y].tar.gz:

  1. grinput.o: código-objeto contendo funções para leitura de arquivos de entrada.
  2. trab_main.o: código-objeto contendo a entrada de dados e a correção do trabalho.
  3. trab[Y].o: código-objeto de gerenciamento do trabalho [Y].
  4. trab[Y]_000000001.c: arquivo a ser preenchido com as implementações das funções especificadas em trab[Y].h (conforme instruções abaixo).
  5. grafalg16.2/trabs/Makefile: script para compilação.
  6. instancias/*.clq: vários grafos usados como instâncias.

O arquivo trab[Y].tar.gz é construído com o comando: 

    tar cvzf trab[Y].tar.gz grafalg16.2/bin/ grafalg16.2/lib/ grafalg16.2/trabs/Makefile grafalg16.2/trabs/autor_0000000001/trab[Y]\_0000000001.c doc instancias/ lib/ datast/heap.h graph/direct/ graph/graph.h

A organização desses arquivos nos diretórios é a seguinte: 

    datast
    -- heap.h

    graph
    -- graph.h
    +- direct
       -- bitmap.h

    grafalg16.2
    +- trabs
    |  -- Makefile
    |  +- autor_0000000001
    |     -- trab[Y]_0000000001.c
    +- lib
    |  +- autor_0000000001
    +- bin

    lib
    -- grinput_32.o
    -- trab_main_32.o
    -- grinput_64.o
    -- trab_main_64.o
    +- trab[Y]
       -- trab[Y]_32.o
       -- trab[Y]_64.o

    doc

    instancias

Os diretórios grafalg16.2/lib e grafalg16.2/bin destinam-se a abrigar códigos gerados como produtos da compilação de trab[Y]_0000000001.c. O diretório doc contém esta documentação. Os arquivos gerados na compilação são o código-objeto grafalg16.2/lib/trab[Y]_0000000001_[ARCH].o e o executável grafalg16.2/bin/autor_[XXXXXXXXXX]_[ARCH].

Preparação

Os passos para a preparação de um trabalho são os seguintes:

  1. Recuperar o arquivo compactado do trabalho, denominado trab[Y].tgz.
  2. Crie um diretório para abrigar os arquivos. Nesta documentação, refere-se a esse diretório pelo símbolo [LABGADIR].
  3. Descompactar o arquivo trab[Y]-tar.gz em [LABGADIR], o que criará uma estrutura de diretórios como a indicada acima.
  4. Renomear o arquivo trab[Y]_0000000001.c, substituindo o 0000000001 pelo seu número de matrícula [XXXXXXXXXX] (somente os números, omitindo o "-").
  5. Renomear os diretórios grafalg16.2/trabs/autor_0000000001 e grafalg16.2/lib/autor_0000000001, substituindo o 0000000001 pelo seu número de matrícula (somente os números, omitindo o "-").
  6. Copiar os seus arquivos trab1_[XXXXXXXXXX].c, ..., trab[Y-1]_[XXXXXXXXXX].c para o diretório grafalg16.2/trabs/autor_[XXXXXXXXXX].
  7. Editar o arquivo trab[Y]_0000000001.c usando um editor de textos a fim de:
    1. Na função ident[Y](), substituir 0000000001 pelo seu número de matrícula (somente os números, omitindo o "-").
    2. Implementar o código de cada uma das demais funções.
  8. Compilar.
  9. Executar.

Programação

As funções a serem implementadas estão listadas no arquivo trab[Y]_0000000001.c. Os detalhes das suas especificações estão disponíveis no link trab[Y].h na aba Arquivos.

Algumas das funções em trab[Y]_0000000001.c podem já estar implementadas. Esses casos devem ser tomados como exemplos de utilização de funções de graph.h e direct/bitmap.h.

Compilação

  1. Editar o arquivo grafalg16.2/trabs/Makefile, alterando as definições das seguintes variáveis: 
     WKSPCDIR:=$(HOME) -->  WKSPCDIR:=$(HOME)/[Caminho até o diretório [LABGADIR]]
 SRCDIR:=$(WKSPCDIR) --> SRCDIR:=$(WKSPCDIR)/[LABGADIR]
  2. Executar o comando make alltrabs no diretório [LABGADIR]/grafalg16.2/trabs.

Caso seja necessário, o comando make clearall quando executado no diretório [LABGADIR]/grafalg16.2/trabs remove todos os arquivos gerados em uma compilação.

Execução e Testes

No diretório [LABGADIR]/grafalg16.2/bin, executar o comando ./autor_[XXXXXXXXXX]_[ARCH]. A primeira ação a ser realizada é a definição do grafo a ser usado, o que é feito na opção g- grafo. A opção e- executar deve ser usada para selecionar o problema e executá-lo.

Realizar testes com os diferentes grafos disponíveis.

Arquivo a Enviar

O arquivo a ser enviado é trab[Y]_[XXXXXXXXXX].c.
